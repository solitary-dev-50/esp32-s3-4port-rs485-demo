# 4-Port RS485 Control Board + Web Demo System

[English README](README.md)

这是一个基于 ESP32-S3 的 4 路 RS485 采集与控制演示项目。

项目目标很直接：用一套可重复验证的最小系统，证明板子的基础能力，包括 RS485 通信、Modbus 继电器控制、温湿度读取、开机自检、本地蜂鸣器测试、RGB 指示灯测试，以及一个本地网页演示控制台。

它不是已认证的工业 PLC，也不是已经面向量产交付的工业控制器。

## 当前包含内容

- RS485 演示板固件
- 本地网页演示控制台
- 固定的已验证 Modbus 设备配置
- 本地指示器和 RS485 设备开机自检
- 串口诊断日志
- 接口说明和开发说明
- RS485/Modbus 常见问题排查文档

## 开源范围

本仓库开放固件、网页控制台、接口文档、排查指南和演示示例，用于学习、集成和二次开发。

初始硬件生产文件不包含在本仓库中，包括原理图源文件、PCB 布局文件、Gerber 文件、BOM 和贴片坐标文件。

## 已验证演示配置

| RS485 端口 | 设备 | 地址 | 已验证功能 |
| ---------- | ---- | ---- | ---------- |
| RS485-1 | Single Relay Tester | 8 | 继电器开关 |
| RS485-2 | 16-Channel Relay Controller | 20 | 1-16 路继电器开关 |
| RS485-3 | Temp & Humidity Sensor 1 | 3 | 温湿度读取 |
| RS485-4 | Temp & Humidity Sensor 2 | 6 | 温湿度读取 |

已整理一份开机自检和正常轮询记录，见 [Verification Log](docs/VERIFICATION_LOG.md)。

开机自检已验证：

```text
[SELFTEST] PASS devices=4/4
```

## 主要功能

- 读取两路 Modbus 温湿度传感器
- 读取继电器状态
- 控制 RS485-1 上的单路继电器
- 控制 RS485-2 上的 16 路继电器模块
- 在本地网页控制台显示传感器数值和继电器控制按钮
- 正常轮询前运行开机自检
- 输出串口日志，便于排查问题
- 触发本地蜂鸣器和 RGB 指示灯测试

## 硬件说明

这块板定位为 RS485 采集与控制的开发和演示平台。

- 4 路独立 RS485 端口
- RS485 A/B 线带 TVS 防护，用于辅助抑制静电和瞬态冲击
- 每路 RS485 均提供 120Ω 终端电阻跳线选择位，可根据总线位置和实际需求通过跳线帽手动启用或关闭终端匹配
- 每路 RS485 使用独立收发链路
- 电源输入路径包含保险丝和肖特基二极管保护设计
- 板上使用大容量电容和局部去耦电容辅助提升电源稳定性

这些保护设计有助于开发和现场测试，但不应被理解为已通过认证的工业级浪涌或静电防护。

## 编译

本项目使用 PlatformIO。

```bash
pio run
```

烧录和串口监控命令取决于本地开发环境和实际连接的板子。

## 安全说明

强电负载应通过接触器或中间继电器控制。本板提供的是控制信号和低压控制接口，不应作为直接驱动强电负载的设备。

开机自检只读取继电器状态，不应自动打开继电器输出。

## 这个项目不是什么

- 不是已认证的工业 PLC
- 不是已完成量产交付的工业控制器
- 不是直接驱动强电负载的设备
- 不是农业专用板
- 也不只是继电器板

## 当前范围

当前版本是固定配置的已验证演示系统。优先目标是保持基线稳定、容易理解、容易复现，再考虑继续扩展功能。

后续可以根据项目需要扩展新的 Modbus 设备、调整网页显示、接入日志历史、增加报警规则，或适配外部设备协议。
