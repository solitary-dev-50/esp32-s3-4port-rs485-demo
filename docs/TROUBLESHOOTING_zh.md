# RS485 / Modbus 避坑指南

适用于 4-Port RS485 Control Board + Web Demo System 的实用排查说明。

这份文档重点讲排查思路和常见坑，不介绍新功能。建议先确认已验证基线能正常跑通，再去调整固件、接线、设备地址或网页显示。

## 1. 当前版本范围

当前版本是固定配置的已验证演示套件。网页端暂不支持动态添加或配置 RS485 设备。

如果项目需要网页配置 RS485 设备，可以作为后续版本或定制服务来做。

## 2. 已验证基线

当前演示基线已经验证以下 RS485 设备：

| RS485 端口 | 设备 | Modbus 地址 | 已验证功能 |
| ---------- | ---- | ----------- | ---------- |
| RS485-1 | Single Relay Tester | 8 | 继电器开关 |
| RS485-2 | 16-Channel Relay Controller | 20 | 1-16 路继电器开关 |
| RS485-3 | Temp & Humidity Sensor 1 | 3 | 温湿度读取 |
| RS485-4 | Temp & Humidity Sensor 2 | 6 | 温湿度读取 |

开机自检已验证：PASS devices=4/4

改任何东西之前，先确认这套已验证基线能正常工作。

## 3. 建议：一次只排查一个变量

排查要一层一层来：

1. 先确认电源。
2. 再确认 RS485 A/B 接线。
3. 再确认接的是正确 RS485 端口。
4. 再确认 Modbus 地址。
5. 再确认功能码。
6. 再确认寄存器或线圈地址。
7. 再确认 CRC 和超时。
8. 最后才改固件。

建议一次只调整一个变量，比如先确认接线，再确认地址，最后再看网页或固件。这样排查路径更清楚，出了问题也更容易定位。

## 4. 问题：RS485 设备没有响应

先查这些项目：

- 设备电源是否正常
- RS485 端口是否接对
- A/B 线是否接反
- Modbus 地址是否错误
- 波特率是否错误
- 功能码是否错误
- 寄存器或线圈地址是否错误
- 需要共地的场景是否没有共地
- 线缆或接插件是否有问题
- 长距离现场接线是否缺少合适终端电阻
- 设备是否离线或损坏

如果日志没有 RX 数据，或者显示超时，通常应先查物理接线和协议参数，再看解析逻辑。

## 5. 问题：CRC 错误

常见原因：

- 干扰
- 接线质量差
- 波特率错误
- 串口帧格式错误
- 预期回包长度不对
- 半双工方向控制问题
- 读取超时时间太短

CRC 错误的意思是：系统收到了东西，但这一帧数据不可信。

## 6. 问题：继电器控制发到了错误设备

继电器控制建议带上明确目标，不建议只传继电器编号和状态。

错误思路：

```text
{ "id": 1, "state": true }
```

正确思路：

```json
{
  "target": "single",
  "id": 1,
  "state": true
}
```

```json
{
  "target": "multi",
  "id": 1,
  "state": true
}
```

后端应把 target 明确解析成 RS485 端口 + Modbus 地址 + 继电器编号。

- single target = RS485-1，addr=8，只允许 relay id = 1
- multi target = RS485-2，addr=20，允许 relay id = 1-16

更稳妥的做法是：继电器目标由 target 明确决定，写入逻辑使用明确的通道和地址。日志里应能看到 target、channel、address、relay id、state 和 result。

示例日志：

```text
[WEB][RELAY] target=single channel=RS485-1 addr=8 relay=1 state=ON success
[WEB][RELAY] target=multi channel=RS485-2 addr=20 relay=16 state=ON success
```

## 7. 问题：传感器能读到，但数值不对

重点查：

- 起始寄存器地址
- 寄存器数量
- 功能码 0x03 和 0x04 是否混用
- 缩放系数
- 有符号还是无符号
- 字节序或字序
- 单位换算

当前已验证演示中的温湿度传感器使用：

- 功能码：0x04
- 起始寄存器：0x0000
- 寄存器数量：2
- 缩放：原始值 / 10.0

如果没有设备协议文档，建议先确认寄存器含义，再扩展新的寄存器定义。

## 8. 问题：网页看起来正常，但设备不动作

检查顺序：

- 浏览器请求体
- target 是否正确
- 后端是否解析 target
- target 是否映射到正确 RS485 端口和地址
- 继电器控制函数是否显式传入 channel 和 address
- 串口日志是否出现 TX
- 是否收到 RX 回包
- CRC 是否正确
- 继电器模块是否实际吸合

如果没有 TX 日志，问题在 RS485 发送之前。如果有 TX 但没有 RX，查接线、地址和设备。如果 RX 和 CRC 都正常但设备不动作，查继电器模块接线或负载侧。

## 9. 问题：桌面测试正常，现场不稳定

当前系统是演示版最小系统，不是已经完成认证的工业现场成品。

现场部署前要评估：

- 屏蔽双绞线
- 正确接地
- 终端电阻
- 偏置电阻
- 浪涌和静电防护
- 隔离
- 外壳
- 电源稳定性
- 线缆是否远离强电线路

对外使用时，建议按演示套件、开发套件或集成基础来描述，不建议把它说成已经完成工业认证的成品控制器。

## 10. 问题：强电负载控制

本板提供控制信号和低压控制接口。强电负载应通过接触器或中间继电器控制，这块板不适合作为直接驱动强电负载的设备。

常见负载：

- 水泵
- 风机
- 照明回路
- 加热器
- 电机
- 压缩机

这些负载应该通过合适的接触器、中间继电器和保护器件控制。

## 11. 开机自检说明

开机自检用于：

- 蜂鸣器测试
- RGB 报警指示灯测试
- RS485 设备在线检查
- 传感器读取测试
- 继电器状态读取测试

开机自检建议只读取继电器状态，不自动打开继电器输出，也不做吸合动作。

示例：

```text
[SELFTEST] PASS devices=4/4
```

## 12. 推荐排查顺序

1. 给板子上电。
2. 确认蜂鸣器和 RGB 自检。
3. 确认 PASS devices=4/4。
4. 检查当前 RS485 端口映射。
5. 检查 Modbus 地址。
6. 检查功能码。
7. 检查 TX 日志。
8. 检查 RX 日志。
9. 检查 CRC 结果。
10. 检查解析结果。
11. 检查网页请求 target。
12. 检查继电器输出或传感器数值。

## 13. 基线稳定前，先别急着扩功能

当前版本建议先作为固定配置的已验证演示套件使用。

基线稳定前，以下能力更适合作为后续版本或定制项：

- 网页配置 RS485 设备
- 动态设备表
- 规则引擎
- 定时任务系统
- 类 PLC 逻辑
- 语音控制
- 云端看板

这样做的好处是边界清楚，客户先拿到一套能跑通、能验证、能继续开发的基础系统。
